Problem

We all have images, folders, documents and much more on our computer. 1.8 billion people use a cloud storage service to store this content. There are multiple cloud storage services available but they have clunky dashboards and lack the ability to easily save content from the web. Most of these services do not offer a way to tag and easily search content.

Solution

Dream Space aims to create an intuitive experience for users who need to gather inspiration, organize their thoughts and content in projects, and collaborate with team members all in one place. The user has the ability to save content (from the web or via uploads), categorize it with tags, and store it into a project where team members can chat and place comments.

Competitive Analysis

An analysis of market competitors Pinterest, Dropbox and Box found that there was an opening in the market for a product that combined the features of Pinterest and Dropbox. Dreamspace has the ability to organize your files visually combined with the collaboration and file storage features of Dropbox.

User Flows

The user stories were turned into user flows. User flows layout how a user would navigate through the platform. Working through the user flows sets up the roadmap for the website. The flow of creating documents, files or folders changed as the design evolved. I learned that it is crucial to map out each user flow so time is not lost redesigning the website.

Wireframes

Low-fidelity wireframes were sketched and tested. The user tests for the lo-fi wireframes were conducted in person, where users were asked to complete a set of tasks such as onbaording, uploading files, and sharing content.

User testing the low-fi wireframes provided valuable feedback that shaped the high-fidelity wireframes. The most significant change was to the onboarding process. The user testing indicated that the onboarding process needed to be simplified.

A usability test script was created and six users tested the hi-fi wireframes. Overall, the users had greater success with the new simpler onboarding process.

The next challenge encountered was the ability to create content and upload files. Users had some trouble finding the correct buttons. These buttons were modified in the next round of mockups.

User Testing

ONBOARDING



  • • Ten missions were completed by each user via Maze.

  • • The first three missions were related to onboarding. All three of these tests had simple one or two step directions.

  • • The flow of screens provided was intuitive and easy to follow.

  • • These missions were completed with close to 100% success.



“very easy and intuitive”

- maze tester #1
FILE UPLOAD



  • • The next seven questions were related to the content creation within Dream Space.

  • •The biggest problem users ran into was completing two or more steps. The users could "create a folder" but they had trouble "creating a folder and uploading files into the folder. "

  • • 33.3% of users gave up when they were not able to figure out how to upload the files.




  • • I changed my design to redirect the user back to the file name and highlight it so that it was higher in the visual hierarchy. This made it clear that the users click into the folder to add files to it.
DASHBOARD



  • • Users were confused by the dashboard. The team member and recent activity information proved unnecessary.

  • • The header "Home" was too low when designing the other pages like the document editor. I needed to find the best placement of the header via user testing.

  • • During user testing I discovered that users did not know that the " +New" button would lead to uploading documents. I had to change the way users created new documents and how they uploaded content.

  • • It was brought to my attention during user testing that the thumbnails were hard to read. The green and black color combination did not work.




  • • User testing prompted the removal of the team members information on dashboard. This improved the readability of the dashboard.

  • • The location of the header was moved to make more space for other features. User testing was conducted to finalize the location of the page header.

  • • The name of the +New button was changed to "Upload" . This helped users find the upload feature quicker. Further user testing reveled that the colors of the buttons needed to change to improve the hierarchy.

  • • The color of the thumbnails was changed to improve readability.

  • • New icons were designed for the dashboard to enhance the brand identity.

  • • A list view options was created to give the user more control over their dashboard.
DOCUMENT EDITOR



  • • The location of the team members was not ideal. Users were confused about it’s function. The team members are apart of the global project so it did not make sense for them to be attached to the document

  • • The chat and comment buttons needed to be refined. The users were confused when asked to make a comment vs. opening a chat.



  • • The format of the logo was changed in the navigation bar to clean up the design.

  • • The location of the team members changed a few times throughout testing. The final location of the team members was decided by a user click test.

  • • The selected navigation bar text changed from highlighted to a secondary color. This enhanced readability.

  • • The chat and comment buttons were altered to simplify the design.
LANDING PAGE









  • • Original designs started the hero section with a more organic curved lines. Once the rest of the content filled in it did not make sense to carry this design through.

  • • Some users had a hard time reading sections.

  • • The pricing section was hard to read and felt too small. Users did not immediately know which option was the best.

  • • New icons were designed for the features section. These provided branding continuity.

  • • The background color of sections were changed to enhance readability. I also added more padding between items.

  • • The pricing section formatting was changed based on user feedback. The new format highlighted the popular option. The new text alignment enhanced readability.

  • • The format of the user testimonials was changed to match the features section and provide more design continuity.

Conclusion

Lessons Learned

  • • The design process is crucial. Do not skip ahead to branding and mockups.

  • • You cannot underestimate user research. Do as much user testing in person as possible. Documenting facial expressions is helpful.

  • • Keep quality records of conversations and testing completed along the way, no matter how small.

  • • User research dictates the design not the designer.

  • • Keep the MVP within scope.

  • • Document why you change any and all designs.

  • • Sketch many versions of wireframes.

  • • If you do the work in the beginning and document it, designing the MVP will be a lot easier. It will cut down on rework. It will also provide documentation to all the work you provide for the client.
